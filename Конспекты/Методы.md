# Методы

В Go нет классов, однако можно добавлять методы к типам:

Метод в Go это функция со специальным аргументом получателеми (`receiver`).

Получатель указывается в собственном списке аргументов между ключевым словом `func` и имененем метода.

В примере ниже получатель имеет тип `Vertex` и имя `v`:
```
type Vertex struct {
	X, Y float64
}

func (v Vertex) Abs() float64 {
	return math.Sqrt(v.X*v.X + v.Y*v.Y)
}

func main() {
	v := Vertex{3, 4}
	fmt.Println(v.Abs())
}

```

Метод - это просто функция с аргументом получателем.

Методы можно объявлять на неструктурных типах:
```
type MyFloat float64

func (f MyFloat) Abs() float64 {
	if f < 0 {
		return float64(-f)
	}
	return float64(f)
}
```

Тип аргумента получателя должен быть определен в том же пакете что и метод, что относится и к встроенным типам, таким как `int`.

В качестве получателя может выступать указатель:
```
func (v *Vertex) Scale(f float64) {
	v.X = v.X * f
	v.Y = v.Y * f
}
```

В таком случае будет изменять X и Y оригинальной переменной v, а не ее копии в случае без использования указателя.

Методы в отличие от функций могут принимать как указатели, так и значения в качестве аргумента, в то время как функции обязаны принимать указатель если в сигнатуре функции указан указатель. В случае с методами Go автоматически использует указатель: `v.method() => (&v).method()`.
